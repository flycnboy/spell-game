一、应用定位与用户

  • 做什么：帮小学生（「梓晏」）练英语单词拼写/听写，家长/运营人员可管理词库与查看统计。
  • 形态：PWA（渐进式 Web 应用），浏览器打开即用，可「添加到主屏幕」伪装在安卓平板/手机上当作 App。
  • 核心体验：听音 → 拼写/听写 → 即时判定 → 看释义 → 下一题 → 星级总结；错词自动进入错题本并可强化练习。
  • 差异化：内置艾宾浩斯遗忘曲线（第 1/7/16/30 天自动排复习），以及「联网取词释义 + 例句」的富化能力。

三、架构与目录结构

目录职责

  • src/App.tsx：根组件，纯「阶段路由」——按 phase 渲染对应页面，并承担跨 hook 的编排（提交答案、跳过的统计落库等）。
  • src/store/gameStore.ts：Zustand store，管理一局游戏的瞬时状态与流转（startGame / submitAnswer / nextWord / quit 等）。
  • src/hooks/*：5 个 hook 各自封装一类「浏览器本地存储」数据，是事实上的数据访问层。
  • src/components/*：11 个页面/面板组件，UI 与交互集中于此。
  • docs/：Vite 构建产物，即 GitHub Pages 实际托管的静态站点。
  • generate_help_doc.py：用 python-docx 生成《操作与运维手册》docx，文档即代码。

四、核心功能拆解

① 两种学习模式

拼写：字母打乱成方块，点选或键盘输入拼出单词。
听写：只放发音（可设 1/2/3 遍），孩子手输单词，答错可「显示答案」。

② 词库管理

远程同步(Gist)、本地新建、JSON 批量导入/导出、词库重命名/删除、单词增删、从其他库导入、勾选另存新库。

③ 艾宾浩斯复习

新词在第 1/7/16/30 天自动排入复习；每日先展示「今日计划」再开练；学完自动推进到第二天。

④ 单词释义富化

联网取中文翻译+英文释义+例句，结果缓存本地，下次免流量；释义在听音/拼写/结果页展示。

⑤ 统计与错词本

总次数/正确率/今日量；错词列表，答对自动移除；一键「强化练习错词」。

⑥ 完成总结

本轮正确率 + 星级（≥90% 三星）+ 逐词清单；可再来一轮或强化错词。

五、数据流与持久化

所有状态都落在浏览器本地存储上，没有服务端。共 5 个键：

键名内容写入方
spellgame_batches全部词库 + 当前词库 iduseWordBanks
spellgame_stats错题本 + 历史答题记录useStats
spellgame_settings每天新词数 / 总天数useStudyPlan
spellgame_enriched单词释义缓存（按词）useEnrich
spellplan_<batchId>当前学到第几天useStudyPlan
数据孤岛风险：本地存储只在本机本浏览器有效。换设备/清缓存/换浏览器，词库以外的数据（统计、释义缓存、学习进度）都会丢失；词库虽可经 Gist 同步，但统计与进度无法跨设备。备份只能靠手动「导出全部」JSON。

六、代码质量评估

优点

  • 分层清晰：UI / 状态 / 存储三层分离，组件职责单一。
  • 状态集中：游戏流转统一在 Zustand，便于推理。
  • 数据访问收敛到 5 个 hook，本地存储读写有统一封装。
  • 移动优先 + 自定义动画，儿童交互友好。
  • 文档齐全：操作手册 docx + 帮助文档生成脚本 + 部署说明。
  • 释义渲染走 React 文本节点，天然防 XSS。
  • 富化失败被 try/catch 吸收，单点 API 故障不致命。

不足

  • 死代码：useSpeech.ts 写好却从未被引用（组件直接用原生 speechSynthesis）。
  • 冗余依赖：isomorphic-git 声明未使用。
  • 模板残留：src/assets/react.svg、vite.svg 及根目录 assets/ 旧构建产物未清理。
  • App.tsx 中直接 useGameStore.setState({mode}) 绕过 store action，混合了编排与状态。
  • 无测试（仅 oxlint），useStudyPlan 的复习计算等核心逻辑缺单测保护。
  • 根目录散落数据样例（wordbacks_*.json 等），与源码混放。

七、发现的问题与风险（按优先级）

级别问题影响
P0听音/听写环节需手动点喇叭才发声；进入 phase 不会自动朗读听力练习的关键一步要多一次点击，低龄用户易卡住；建议进入 listen 自动播 1 遍
P0「强化错词」结束后仍会 advanceDay，把复习误当「新的一天」推进计划复习与「学新词进度」耦合，长期会让艾宾浩斯排期失真
P0数据仅存本地，跨设备/清缓存即丢失（统计、进度、释义缓存）换平板或从手机换设备，学习成果清零
P1释义依赖 dictionaryapi.dev（仅英文词条）+ MyMemory，可能 404 或限流个别词释义为空；MyMemory 有软性日限额
P1enrichWords 串行逐个 fetch + 300ms 延迟，大词库耗时长数百词时等待明显（已做每词落盘防中断丢失，这点做得好）
P1「再来一轮」与「返回首页」复用同一回调，都会推进天数仅看总结也想回首页时，天数被+1
P2听写模式只做了小写精确匹配，无近音/拼写容错孩子拼写接近但差一个字母直接判错，体验偏硬
P2无深色模式/大字体等儿童护眼设置；user-scalable=no 禁缩放长时间使用舒适度一般

八、改进建议路线图

立即做（体验修复，成本低）

  • 进入 listen 阶段自动朗读一次（用已有 gUtterance 机制即可）。
  • 复习（review）流程不调用 advanceDay，与「学新词」进度解耦。
  • 清理死代码：删 useSpeech.ts、isomorphic-git 依赖、模板 svg、根 assets/ 旧产物。
  • 把 App.tsx 里直接 setState 收敛为 store 的 action（如 setMode）。

短期做（质量与可靠性）

  • 为 useStudyPlan（复习计算）、gameStore（流转）补单元测试（Vitest）。
  • 释义获取增加超时 + 失败重试 + 降级提示，避免静默空白。
  • 听写增加「大小写/尾标点容错」「近音词提示」。
  • 数据样例（json）移出仓库根或加入 .gitignore。

中期做（能力扩展）

  • 跨设备同步：复用 Gist 把 stats / enriched / 进度也云端化，或接入轻量 BaaS。
  • 支持手动录入中文释义（不依赖联网），离线可用。
  • 错题按「错误次数」加权复习，而非固定 1/7/16/30 间隔。
  • 儿童护眼：深色模式、字号调节、学习时长温和提醒。
  • CI 自动构建 + 部署，免去手动镜像同步（GitHub Actions 直发 Pages）。
